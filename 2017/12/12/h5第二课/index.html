新增的表单校验

  1. checkValidity()检查input元素是否满足验证要求

  2. 特殊需求

js自定义验证方法
使用input元素的setCustomValidity()自定义信息的提示内容

  1. 使用ValidateState对象进行表单验证

DOM接口ValidityState代表了一个元素可以处于的与约束验证相关的有效性状态。
Validity对象是通过validity属性获取的，该对象有8个属性，分别针对8个方面的错误验证，属性值均为布尔值。
3.1 valueMissing属性（必填的表单元素值为空）
如果表单元素设置了required特性，则为必填的值。如果必填的值为空，valueMissing属性返回true，否则返回false.
3.2 typeMismatch属性（输入值与type类型不匹配）
H5新增的表单类型如email number url等，都包含一个原始的类型验证。如果用户输入的内容与表单类型不符合，则返回TRUE否则返回FALSE。
3.3 patternMismatch属性（输入值与pattern特性的正则不匹配）
表单元素可通过pattern特性设置正则表达式的验证模式。若不符合返回TRUE否则FALSE
3.4 tooLong属性（输入的内容超过了表单元素的maxLength特性限定的字符长度）
3.5 rangeUnderflow属性（输入的值小于min特性的值）
3.6 rangeOverflow属性（输入的值大于max特性的值）
3.7 stepMismatch属性（输入的值不符合step特性所推算出的规则）
3.8 customError属性（使用自定义的验证错误提示信息）

HTML文件处理

File API

File API是文件处理规范的基础，包含最基础的文件操作的JavaScript接口设计。其中最主要的接口定义一共有4个：

  1. FileList 代表一组文件的js对象
  2. Blob 用来代表一段二进制数据，并且允许我们通过js对数据以字节为单位进行切割。size type slice() close()
  3. File 代表一个文件，是从Blob接口继承而来的，并在此基础上增加了诸如文件名、MIME类型之类的特征。
  4. FileReader 用来读取文件的方法和事件abort() readAsBinaryString(file)
    FileReader的实例拥有四个方法，其中三个用于读取文件，一个用来中断读取。
    4.1 abort none 中断读取
    4.2 readAsBinaryString file 将文件读取为二进制码
    4.3 readAsDataURL file 将文件读取为DataURL
    4.4 readAsText file,[encoding] 将文件读取为文本

事件包括onabort onerror onload onloadend onloadstart onprogress
