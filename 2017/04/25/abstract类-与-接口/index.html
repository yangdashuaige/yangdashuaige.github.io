abstract类 与 接口

发表于 2017-04-25 | 分类于 Java

首先讲的是抽象类
当编写类时，常常会为其定义一些方法，用来实现某些功能，那么这些方法都有具体的方法体。但在某个条件下，父类只是知道其子类该包含怎样的方法，但无法准确知道子类该如何实现他，这时就需要定义一个抽象类的方法。抽象类与抽象方法用abstract修饰，有抽象方法的只能被定义为抽象类，而抽象类中可以没有抽象方法。如下是抽象类的规则：
抽象类必须使用abstract修饰，抽象方法也是。抽象方法不能有方法体。 抽象类不能实例化，无法使用new关键字来调用构造方法。即使抽象类中不包含抽象方法，也不能创建实例。
抽象方法的构造方法主要用于子类的调用。 含有抽象方法的类只能被定义为抽象类。
抽象类的作用：它体现了一种模板模式的设计，抽象类作为作为多个子类的通用模板，子类在抽象类的基础上进行扩展改造，但子类总体上会大致保留抽象类的行为方式。
模板模式的规则：抽象父类只定义需要使用的方法，其余则留给子类去实现。父类中可能包含需要调用其它系列方法的方法，这些被调用方法既可以由父类实现，也可以由其子类实现。
