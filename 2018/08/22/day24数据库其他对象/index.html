多表连接

-- 1. 外表连接，当两张表没有主外键，这时想要两张表的信息绑定在一起。即使没有主外键关系也可以绑定在一起。
-- 查询员工号为7369的员工所在的部门的名称
select emp.empno,emp.ename,dept.dname from emp,dept where emp.deptno=dept.deptno and emp.empno=7369;

-- 左外连接
select e.ename,d.dname from emp e left join  dept d on e.deptno=d.deptno

-- 右连接
select e.ename,d.dname from dept d left join  emp e on e.deptno=d.deptno
-- 自连接
-- 查询每个员工领导的姓名，没有就是null，自己查自己。
select e1.ename 员工,e2.ename 领导 from emp e1,emp e2 where e1.mgp=e2.empno
-- 查询财务部门里面员工的姓名和工资
select e.ename,e.sal from emp e ,dept d where e.deptno=d.deptno and d.dname='ACCOUNTING'
select e.ename ,e.sal from emp e LEFT JOIN dept d on e.deptno=d.deptno and d.dname='ACCOUNTING'
-- 查询每个部门的平均工资，显示的是部门名。平均工资
select d.dname,AVG(sal) 平均工资 from emp e,dept d where e.deptno=d.deptno GROUP BY e.deptno
select d.dname,AVG(sal) 平均工资 from emp e LEFT JOIN dept d on e.deptno=d.deptno GROUP BY e.deptno
-- 查询部门地址为null部门里的员工的个数
select dept.dname 部门, count(*) 部门总人数 from emp,dept where dept.deptno=emp.deptno and dept.loc='NEWTORK';
select dept.dname 部门, count(*) 部门总人数 from emp LEFT JOIN dept on dept.deptno=emp.deptno where  dept.loc is null group by dept.dname
-- 查询比史密斯工资高的员工的姓名和工资。（子查询）首先要确定史密斯的工资
select ename,sal from emp where sal>(select sal from emp where ename='SMITH')
-- 上述的过程称为子查询



-- 查询平均工资低于4000的部门及其员工信息
SELECT * from emp where deptno in(select deptno from emp GROUP BY deptno HAVING AVG(sal)<4000 )

数据库中的其他对象

索引：提高查询效率，给列的字段建立索引

注意点：经常变动的列不建议建立索引。
create index job_index on emp(job)

视图：是一张虚表：安全

是依赖于具体表提取出来的部分数据，视图一般不建议修改，只能查看。

给emp这张表创建一个视图，全部字段

create index job_index on emp(job)

create view vemp as (select * from emp)
select * from vemp


-- 查询job为clerk员工的全部信息
select * from vemp where job='CLERK'

事务：多个数据库操作语句，要么全部成功，要么全部不成功。

  1. 原子性：要么全部成功，要么全部不成功
  2. 隔离性：一个事务的操作不影响另一个事务
  3. 一致性：数据的结果是一致的，转账前总和和转账后总和一致。
  4. 持久性：只要事务commit以后，永久保存不回退

数据库语言

  1. DCL：定义语言 create ,drop ,alter, truncate 自动提交，运行后永久保存结果，不可回退
  2. DML：操作语言，insert update delete需要手动提交的，如果事务不提交，会回滚。mysql默认是开启自动提交的事务，关闭该功能

mysql的隔离级别：

  1. Read uncommitted 读未提交的事务；可以读到脏数据。读到中间状态的数据
  2. Read commit 读已提交的数据。可以提交的数据（默认）：在同一个书屋里读到另一个客户端修改以后提交的数据
  3. Read repeat： 读重复数据，幻读。在同一个事务里读到另一个客户端，增加和修改提交数据
  4. Serializable:串读。只有提交后才能读到另一个线程修改或者删除的数据，
