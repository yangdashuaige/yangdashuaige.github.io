数组为底层

  1. 使用数组可以将一组元素作为一个整体来处理
  2. 缺陷：需要空间连续并且数组在实例化时需要长度，那么内存空间可能长时间浪费。有序线性结构，数据结构单一
  3. 改进： 数据不存在一起（物理分离），没有的数据不开辟空间。

集合的优势

  1. 长度动态扩展
  2. 数据结构丰富：List Set Map
  3. 提供丰富的API，开发效率高

ArrayList

底层用数组实现，查询效率高，增伤效率低，线程不安全

Linkedl

底层用双循环链表实现，查询效率低，增删效率高。

底层的增删改查的时间复杂度，什么时候用ArrayList，什么时候用LinkedList。

  1. ArrayList的控件复杂度比LinkedList，存在控件浪费
    1. 如果查询比较频繁，使用ArrayList
    2. 如果添加和删除比较频发，LinkedList，当数据量大时，用LinkedList。

List总结

  1. List是Collection的子接口
  2. 存放的元素是有顺序的
  3. 存放的元素是可以重复的
  4. 存放的元素都有一个整型的下标，可以根据下标访问元素

Set

public interface Setextends Collection
一个不包含重复元素的collection，不能根据位置来获得数据元素

如何将自定义的类对象存入HashSet进行去重？

类必须重写hashCode()方法，equals()方法。

TreeSet底层是红黑树：平衡二叉排序树。集合中的元素没有顺序，也不能重复，会自然排序。

特点：左孩子比跟小，右孩子比根大

TreeSet存放的数据一定要有比较算法

  1. 比较算法：自然排序比较算法放在实体类里面，实体类必须实现Comparable接口
  2. 比较器排序：排序算法以形参的实现传递给TreeSet容器，new Compator接口实现类对象

Comparable接口

此接口强行对实现它的每个类的对象进行整体排序。这种排序称为类的自然排序，类的compareTo方法被称为它的自然比较方法。
