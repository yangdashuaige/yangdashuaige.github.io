java默认的包是java.lang java语言包。所以该包下的所有的类不需要导包，其他包下的所有类使用需要导包。

Object类

Object类是类层次的根类，每个类都使用Object作为超类，所有对象（包括数组）都实现这个类的方法。子类都可以重写父类的方法，那么意味着任何类都能继承或者重写Object的方法（final，private除外）通过重写toString 方法，可以改变输出的格式，得到属性。

equals()和==

  1. equals()是个方法，只有对象才能调用，8个基本数据类型没有
  2. equals和==默认是一个意思
  3. ==已经是比较地址了，所以再有一个equals方法有点重复，能不能两个方法的意义不一致。一般重写equals方法，让它不比较地址而是比较属性。

protected void finalize()

当垃圾回收期确定不存在该对象的更多引用时，有对象的垃圾回收器调用此方法。垃圾回收期回收对象时，会调用该对象的finalize方法，做一些资源的释放。程序员无法控制垃圾回收器，只能提醒。

String类

String创建的是常量，一旦创建，不可改变。字符串存在字符串常量池中，一旦开辟空间，值不发生改变。变量的拼接，先开辟空间，再拼接

大小写的问题

  1. boolean equals(Object anObject)将此字符串与指定对象进行比较
  2. boolean equalsIgnoreCase(String anotherString)将此String与另一个String比较，不考虑大小写
  3. String toUpperCase()使用默认语言环境的规则将String中所有的字符都转换为大写
  4. String toLowerCase()使用默认语言环境的规则将String中所有的字符都转换为小写

String 字符常量，一旦定义不能发生改变，所以在字符串反复变化拼接的时候，对于内存的开辟和销毁操作频繁，如果存储的数据经常变动，不建议使用String，对于内存的销毁比较频繁。

String、StringBuffer和StringBuilder区别

String是字符串常量，一旦定义内容不变，StringBuilder、StringBuffer可变字符序列。StringBuilder线程不安全，Stringbuffer线程安全。
