解答

  1. 面向对象
  2. 啦啦啦

  3. 区别：

    • 类中的位置不一样
    • 内存位置不一样：栈 、堆
    • 生命周期：方法的生命周期一致，对象的生命周期一致
    • 初始值：局部变量必须赋初始值，而属性可以不赋（JVM帮忙赋值）

  4. 类的加载》执行静态代码块》new》堆里面开辟空间》调用构造代码块和构造方法
    5.单例模式

    1. 只有一个实例
    2. 构造方法私有化(让外部无法实例化)
    3. 本类私有的静态的属性（自己内部调用构造私有的构造方法实例化，在类加载的时候创建该对象，以后再也不创建了，static，为了防止用户修改，设置private）
    4. 给属性设置getter（为了保证用户获得，get方法）

  5. 区别：

    1. 在类中的位置：都在类中，类变量是static修饰的
    2. 在内存中的位置：类变量在方法区的静态池，属性在堆中
    3. 生命周期：与类一致，与对象一致
    4. 调用的方式：类名.*;对象名.

  6. 不能，生命周期不对

  7. 静态代码块，构造代码块，局部代码块
  8. 在一个类中，方法名相同，参数类型或者个数不一样
  9. 如果一个包中用到另一个包中的类，那么需要将该类导入
  10. 构造方法：无返回值，方法名与类相同，没有jvm会添加默认的，对象实例化的时候调用

继承

孩子享受父亲所有的特性(private)，孩子会优于父亲，父亲无法享受孩子的特性。

访问权限：protected 受保护的，同包或异包子类可以访问吗，同一包中其他类也能访问

this,super和static都不能共存。this代表本类对象，相当于我的意思，super代表对象的父亲，相当于我的父亲。static是类，抽象的，与this和super的具体对象是冲突的，不能共存。

一个对象那个的实例化的过程

父亲有静态代码块，有构造代码块，有构造方法。孩子有静态代码块，有构造代码块，有构造方法。

首先是父亲的静态代码块》孩子的静态代码块》父亲的构造代码块》父亲的构造方法》孩子的构造代码块》孩子的构造方法

super（） 与this（）不共存。

重写

子类对父类中的方法重写。

重写规则：两同两小一大

  1. 两同：方法名相同，参数类型也必须相同
  2. 两小：返回类型必须是父亲返回类型的子类或者本身；子类抛出的异常必须是父类抛出异常的子类或者相同
  3. 一大：访问权限，孩子的访问权限一定大于等于父亲的访问权限

访问权限：public（所有都可以访问） 、protected（除了同包下的，只有异包子类）、默认（只有同包）、private（私有）

重载：同一个类，方法名相同，参数的个数和参数的类型不同。

Final

最终的，不可改变。修饰变量：变量值不发生改变。
修饰类：不能被继承。修饰方法：修饰的方法不能被重写。

多态

父类引用指向子类对象，以继承为基础。方法的编译看左边，如果父类没有这个方法，那么方法不能调用。方法运行看右边
