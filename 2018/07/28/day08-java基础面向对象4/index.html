继承和多态总结

  1. 孩子实例化的时候，会先实例化父亲，再实例化自己。孩子实例化调用自己的构造方法，孩子的构造那个里面会隐含super（），调用父亲的构造方法实例化对象，默认是无参的，可以修改调用父亲有参的super(参数)
  2. 继承的关键字extends，会继承父亲所有非private的方法和属性，并且子类可以重写父类的方法，重写的原则是：两同（方法名和参数）两小（返回类型子类，异常子类）一大（访问权限大）原则
  3. 如果孩子直接继承父亲的属性，那么在给孩子属性赋值的时候，父亲的属性也一并被赋值，如果孩子覆盖了父亲的属性（孩子的属性和父亲的同名），那么久不会给父亲属性赋值
  4. this代表自己 ，也就是我，super代表父类的对象，子类的构造方法的第一句话要么是this（），要么是super（），两者不同时出现。

方法

  1. 抽象的方法，有大括号，做事情
  2. 具体的方法，做事情

如果子类的父类是抽象类，那么必须重写抽象方法

Java为了降低类与类之间的耦合度，java提倡的模式：依赖于抽象不要依赖于具体。

耦合度：在开发阶段

抽象类

  1. 具有抽象方法的类叫做抽象类
  2. 抽象类里面可以有抽象方法，也可以没有抽象方法，普通类里面的东西都可以有
  3. 属性，构造方法，构造代码块，普通方法
  4. 抽象类和普通类的区别：有抽象方法的类一定是抽象类
  5. 抽象类不能实例化对象，构造方法是给子类使用的
  6. abstract与FInal不能共存。继承的子类必须重写父类的抽象方法，所以抽象方法不能是abstract和private
  7. 如果子类不重写父类的抽象方法，那么子类也还是抽象的类
  8. static与abstract也不共存
  9. 抽象类就是过度接口和具体类

接口

  1. 完全抽象，接口中不能全部是抽象方法（jdk1.8以后可以有具体方法的）
  2. 访问权限必须是public，接口的抽象方法都是public abstract的。
  3. 接口没有构造方法，不能实例化对象
  4. 接口不建议有属性，如果有一定是public static final修饰的
  5. 接口像大纲：操作类

接口和抽象类的区别：

  1. 里面的内容：接口是完全抽象，只能有抽象方法（public abstract），没有构造方法，属性是public static final修饰的；抽象类可以有构造方法，构造代码块，可以有普通方法，可以有抽象方法，可以有属性，
  2. 两者都不能实例化对象，但是抽象类的构造方法可以给子类使用

  3. 通过多态的方式实例化对象：

    1. 父类引用指向子类对象
    2. 接口引用指向实现类对象

  4. java是类单继承，多实现（类实现多个接口）多实现弥补了单继承的缺陷

  5. 接口可以多继承

二维数组

java原则上是不存在多维数组的，但是当数组的每一个数据类型还是一个数组，那么就是多维数组
