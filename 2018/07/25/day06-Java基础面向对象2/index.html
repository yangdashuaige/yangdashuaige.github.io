day06-Java基础面向对象2

发表于 2018-07-25 | 分类于 洗心革面学Java

复习题

  1. OPP和OOP的区别
  2. 类和对象的区别
  3. setter和getter方法的作用
  4. toString方法的作用
  5. 简述对象实例化的过程
  6. 什么是函数重载
  7. 不定参数的格式
  8. 构造方法的要求及作用
  9. 数组的特点
    • 存储多个相同数据类型的元素
    • 数组定长

回答

  1. OPP：面向过程开发，程序和程序运行过程一致，从上至下分析业务编写代码；OOP：面向对象开发，提取能够完成业务功能的对象，然后相互合作
  2. 类是抽象的模板，对象是类的个体，类的实体
  3. setter设置属性值，getter获得属性值。安全合法化
  4. 如果输出对象名，隐藏地调用对象的toString（）方法，编写toString方法去返回每个属性值，可以按照要求格式输出
  5. 通过new 关键字，在内存的堆中开辟空间存储（根据对象的属性），调用构造方法去给属性初始化
  6. 方法必须属于同一个类，方法名相同，参数的类型或个数不同。
  7. 参数的类型一致，只是参数的个数不定 （类型）…底层是数组
  8. 没有返回值，方法名与类同名。new对象调用，如果没有构造，JVM会添加无参的构造方法，如果有构造方法，就不构造

变量的作用域

一个标准的实体类：属性，方法，setter和getter方法，构造方法，toString。

  • 属性是在对象实例化的时候开辟空间，只开辟跟对象相关的属性，所以我们将属性称为对象的变量，随着对象的销毁而销毁。
  • 方法的变量称为局部变量，调用方法而加载，方法结束而销毁。
  • 类的变量，即静态变量，是类所有，在方法区的静态池，是所有对象的属性。（静态的东西一般通过类名来调用）

关键字static

  • Static代表的类所有，静态的，调用的方法是 类名.*。
  • Static可以修饰属性，可以修饰方法，可以修饰代码块。修饰的属性就是类的变量，修饰的方法就是类的方法，调用的时候 类名.*。public static String sex;
  • 静态的修饰的变量是在类加载的时候初始化，类销毁的时候才销毁
  • 生命周期，JVM先将字节码文件全部加载，碰到静态变量，就在静态池中开辟空间存储。
  • 静态的方法只能调用静态的变量和方法，不能调用非静态的
  • 非静态的既可以调用的静态的也可以调用非静态的。

代码块

构造代码块的：在执行构造方法的时候，会先调用构造代码块，然后调用构造方法。它的作用是为了编写构造方法的重复代码。

局部代码块

代码在方法里，一般不会用到

静态代码块

静态代码块只执行一次，在类加载的时候执行。顺序：静态代码块》构造代码块》构造方法。、

一个实体类实例化对象的时候，首先是类的加载，执行静态代码块，new关键字开辟空间之后，执行构造代码块和构造方法。

对象实例化过程

new 堆里开辟空间，然后嗲用构造代码块，然后调用相应的构造方法

设计模式

  • 设计模式：就是前辈总结的一些开发方式，开发思想
  • 多对象合作，低耦合，高内聚
  • 单例模式：最多只允许一个实例（单例模式），针对的是操作类
