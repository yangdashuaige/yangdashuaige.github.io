第三方登录

发表于 2018-09-24 | 分类于 项目相关

OAuth协议

在以Web服务为核心的云计算时代，授权的需求变得日益迫切和兴盛，开放授权也因此而生，意在帮助人们将资源授权给第三方应用，支持细粒度的权限控制，并且不会泄露其他认证的凭据。

  1. 使用OAuth协议。现场授权，在线授权
  2. IAM服务。预先授权，离线授权

简单流程

当用户点击QQ登录，你会引导用户跳转到第三方登录的登录界面，当用户成功登录以后会自动跳转回来，并附带一个code参数，接着你使用code就可以获得access token，对于QQ，由于第三方登录结合了OpenID，所以需要通过token获取openid.这个openid是和QQ用户一一对应的，使用access token +openid即可获取用户信息。

6个步骤

  1. 用户打开客户端以后，客户端要求用户给予授权
  2. 用户同意给予客户端授权
  3. 客户端使用上一步获得的授权，向服务器申请令牌
  4. 认证服务器对客户端进行认证后，确认无误，同意发放令牌
  5. 客户端使用令牌，向资源服务器申请资源
  6. 资源服务器确认令牌无误吗，同意向客户端开放资源

客户端的授权模式

  1. 授权码模式：功能最完整，流程最严密的授权模式。特点是通过客户端的后台服务器，与服务提供商的认证服务器进行互动。
  2. 简化模式
  3. 密码模式
  4. 客户端模式

第三方平台和本地用户系统结合的三种情况：

  1. 本地未登录，第一次登录第三方：此时相当于注册，直接把第三方信息拉取来并注册成为本地用户就可以了，并建立本地与第三方用户的绑定关系。
  2. 本地未登录，再次登录第三方：此时用户已经注册，获取到openid后直接找出对应的本地用户即可
  3. 本地登录，并绑定第三方：这个只要将获取到的openid绑定到本地用户即可。
