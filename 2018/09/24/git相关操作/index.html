git基本操作

本地库初始化

进入工作目录，git bash here。mkdir新建一个目录，在目录里面初始化：git init。ls -lA显示隐藏目录。

  1. 命令：git add
  2. 效果：
  3. 注意：ll .git/可以看见目录中存放的本地库相关的子目录和文件，不要删除和修改。

设置签名：

  1. 形式：用户名：tom Email地址：xxx@xx.com
  2. 作用：区分不同开发人员
  3. 辨析：这里设置的签名和等哦路远程仓库的账号密码没有任何关系
  4. 命令：
    1. 项目、仓库级别：仅在当前本地仓库范围内有效git config user.name tom。git config user.email xxx@xx.com
    2. 系统、用户级别：登录当前操作系统用户范围 git config –global user.name tom。git config –global xxx@xx.com

基本命令

  1. git status 工作区和缓存区的状态
  2. git add filename 加入缓存区
  3. git commit -m “commit message” filename 提交缓存区的文件
  4. git log 查看历史记录操作.git log –pretty=oneline;git log oneline;git reflog

  5. 前进或者后退版本。通过HEAD指针移动。

    1. 基于索引值git reset –hard bb9e8c6。
    2. 基于^符号。只能往后不能往前git reset –hard HEAD^^几个代表回去几部
    3. 基于~符号 git reset –hard HEAD~n

  6. git diff filename 比较问价差异。

    1. 将工作区的文件和暂存区的文件进行比较。git diff [文件名]
    2. git diff HEAD/本地库中历史版本 [文件名] 。将工作区和本地库历史记录比较
    3. 不带文件名比较多个。

git分支管理

  1. 什么是分支：使用多条线同时推进多个任务
  2. 分支好处：同时并行开发多个功能，提高效率；如果有分支开发失败，不会对其他分支有影响，删除即可。
  3. 分支操作
    1. 创建分支 git branch [分支名]
    2. 查看分支 git branch -v
    3. 切换分支 git checkout [分支名]
    4. 合并分支 首先切换到接收修改的分支上，git checkout [分支名];第二步，执行merge命令，git merge [有新内容的分支名]
    5. 解决冲突。对于分支的修改，如果该的是同一个文件同一位置，这时git就拿不定主意，以冲突的形式展现出来。
      1. 第一步编辑文件，删除特殊符号
      2. 把文件修改到满意程度，保存退出
      3. git add文件名
      4. git commit不能带文件名

创建远程仓库

  1. 在本地创建远程仓库别名 git remote add origin
    https://github.com/yangdashuaige/test.git
  2. 将本地的信息推送到远程仓库git push origin master

  3. 克隆。git clone https://github.com/yangdashuaige/test.git

    1. 完整的把远程的库下载到本地
    2. 创建origin远程地址别名
    3. 初始化本地库

  4. 拉取

    1. pull=fetch+merge
    2. git fetch 远程仓库别名 远程分支名
    3. git merge 远程仓库地址别名 远程分支名
    4. git pull 远程地址别名 远程分支名

  5. 解决冲突

    1. 如果不是基于GitHub远程库最新版做的修改，必须先拉取。
    2. 拉取下来如果进入冲突状态，按照分支冲突解决操作即可
