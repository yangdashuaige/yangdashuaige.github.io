发表于 2018-11-27 | 分类于 面试

集合类*

  1. 基本介绍：

    1. Collection接口是List，Set，Queue的父接口
    2. Set接口有两个常用的实现类：HashSet和TreeSet
    3. List接口常用接口有：ArrayList和Vector
    4. Map接口有两个实现类：Hashtable和HashMap

  2. Collection实现比较要实现什么接口

    1. 实体类实现Comprable接口，并实现compareTo(T t)方法。
    2. 创建一个外部比较器，这个比较器实现Comparator接口的compare(T t1,T t2)

所以，当希望实体类放入集合的时候能够按照你希望的方式排序，那么需要让实体类实现Comparable接口，。如果只是简单比较两个实体类的大小，最后返回一个结果，那么用Comparator接口实现一个外部比较器比较合适。

  1. ArrayList和Vector的区别（是否有序，是否重复，数据结构，底层实现）

    1. ArrayList和Vector都实现了List接口，他们都是有序集合，并且存放的元素是允许重复的。他们的底层都是通过数组来实现的，因此这种数据结构检索数据速度快，但增删改速度慢。
    2. 区别：
      1. 线程安全。Vector是线程安全的，而ArrayList是线程不安全的。因此如果集合数据只有单线程访问，那么使用ArrayList可以提高效率。而如果有多线程访问你的集合数据，那么就必须使用Vector，因为要保证数据安全。
      2. 数据增长。ArrayList和Vector都有一个初始的容量大小，当存储他们里面的元素超过了容量时，就需要增加它们的存储容量。ArrayList每次增长原来的0.5倍，而Vector增长原来的一倍。ArrayList和Vector都可以设置初始空间的大小，Vector还可以设置增长的空间大小，而ArrayList没有提供增长控件的方法。

  2. HashMap和Hashtable的区别

    1. HashMap和Hashtable都实现了Map接口，并且都是key-value的数据结构。他们的不同点主要在三个方面
    2. 第一，Hashtable是java1.1的一个类，它基于陈旧的Dictionary类。而HashMap是java1.2引进的Map接口的一个实现。
    3. 第二，Hashtable是线程安全的，也就是说线程同步的，而HashMap是线程不安全的。也就是说在单线程环境下使用HashMap，这样效率高
    4. HashMap允许将null值作为key或value，但Hashtable不允许（抛出NullPointException）

  3. List和Map的区别（数据结构，存储特点）

    1. 主要从两个方面回答，一方面是List和Map的数据结构，另一方面是存储数据的特点。
    2. 数据结构：List存储的是单列数据的集合；而Map存储的是key,value类型的数据集合。
    3. 数据存储：List存储的数据是有序的并且可以重复，而Map中存储的数据是无序的且key值不能重复。

  4. List，Map，Set三个接口，存取元素时，各有什么特点？

    1. List和Set具有相似性，他们都是单列元素的集合，他们有一个共同的父接口，叫Collection。Set里面不允许有重复的元素，所谓重复，即不能有两个相等（注意，不是仅仅是相同）的对象。Set取元素时，没法说取第几个，只能以Iterator接口所取得的所有元素，再逐一遍历各个元素
    2. List表示有先后顺序的集合，注意，不是那种按年龄、按大小、按价格之类的排序。当我们多次调用add(Obj e)方法时，每次加入的对象就像火车站买票有排队顺序一样，按先来后到的顺序排序。有时候，也可以插队，即调用add(int index,Obj e)方法，就可以指定当前对象在集合中的存放位置。一个对象可以被反复存储进List中，每调用一次add方法，这个对象就被插入进集合中一次，其实，并不是把这个对象本身存储进了集合中，而是在集合中用一个索引变量指向这个对象，当这个对象被add多次时，即相当于集合中有多个索引指向了这个对象，如图x所示。List除了可以以Iterator接口取得所有的元素，再逐一遍历各个元素之外，还可以调用get(index i)来明确说明取第几个
    3. Map与List和Set不同，它是双列的集合，其中有put方法，定义如下：put(obj key,obj value)，每次存储时，要存储一对key/value，不能存储重复的key，这个重复的规则也是按equals比较相等。取则可以根据key获得相应的value，即get(Object key)返回值为key 所对应的value。另外，也可以获得所有的key的结合（map.keySet()），还可以获得所有的value的结合（map.values()），还可以获得key和value组合成的Map.Entry对象的集合（map.entrySet()）。
    4. List 以特定次序来持有元素，可有重复元素。Set 无法拥有重复元素,内部排序。Map 保存key-value值，value可多值。

多线程

  1. 什么是线程：线程是操作系统能够运行的最小单位，它被包含在进程中，是进程实际操作的单位。程序员可以通过它进行多处理器编程。

  2. 线程和进程的区别：

    1. 线程是进程的子集，一个进程可以有多个线程，每条线程并行执行不同的任务。
    2. 不同的进程使用不同的内存空间，而所有的线程共享一片相同的内存空间。

  3. 多线程的实现。

    1. 使用Thread类：在Thread子类覆盖的run方法中编写运行代码
    2. 实现Runnable接口

JVM内存模型*

  1. JVM内存分哪几个区，每个区的作用是什么？
    1. 方法区：
      1. 有时候也称为永久区，在该区很少发生垃圾回收，但是并不代表不发生GC，在这里进行的GC主要是对方法区里的常量池和对类型的卸载。
    2. 虚拟机栈（Java栈）：java栈是java方法执行的内存模型。
      1. java栈中存放的是一个个的栈帧，每个栈帧对应一个被调用的方法，在栈帧中包括局部变量表（Local Variables），操作数栈（Operand Stack）、指向当前方法所属的类的运行时常量池的引用、方法返回地址和一些额外的附加信息。当线程执行一个方法时，就会随之创建一个对应的栈帧，并将建立的栈帧压栈。当方法执行完毕之后，便会将栈帧出栈。因此可知，线程当前执行的方法所对应的栈帧必定位于java的顶部。
    3. 本地方法栈
    4. 堆
    5. 程序计数器：也称作PC寄存器。用来指示执行哪条指令的。由于在JVM中，多线程是通过线程轮流切换获得CPU执行时间的，因此，在任意具体时刻，一个CPU的内核只会执行一条线程中的指令，因此，为了能够使得每个线程都在线程切换后能够恢复在切换之前的程序执行位置，每个线程都需要有自己独立的程序计数器，并且不能够相互干扰，否则就会影响到程序的正常执行次序。因此，可以这么说，程序计数器是每个线程私有的。在JVM规范中规定，如果线程执行的是非native方法，则程序计数器中保存的是当前需要执行的指令的地址；如果线程执行的是native方法，则程序计数器中的值是undefined。由于程序计数器中存储的数据所占的空间的大小不会随程序的执行而发生改变，因此，对于程序计数器是不会发生内存溢出现象的。

数据库

  1. 事务的四大特性

    1. 原子性：指事务包含的所有操作要么全部成功，要么全部失败回滚。
    2. 一致性：是指事务必须使数据库从一个一致性状态变换到另一个一致性状态，也就是说一个事务执行之前和执行之后都必须处于一致性状态。
    3. 隔离性。是当多个用户并发访问数据库时，比如操作同一张表时，数据库为每一个用户开启的事务，不能被其他事务所干扰，多个并发事务之间要相互隔离。
    4. 持久性：是指一个事务一旦被提交了，那么对数据库的改变就是永久性的，即便是在数据库系统遇到故障的情况下也不会丢失提交事务的操作。

  2. 如果不考虑事务的隔离性，会发生的几种问题：

    1. 脏读：是指在一个事务处理过程里读取了另一个未提交的事务中的数据。
    2. 不可重复读：指在对于数据库中的某个数据，一个事务范围内多次查询却返回了不同的数据值，这是由于在查询间隔，被另一个事务修改并提交了、
    3. 幻读：是事务非独立执行时发生的一种现象。例如事务T1对一个表中所有的行的某个数据项做了从’1’修改为’2’，这时事务T2又对这个表中插入了一行数据项，而这个数据项的数值还是1并且提交给数据库。

  3. 事务的隔离级别：

    1. 读未提交 read-uncommitted
    2. 不可重复读 read-committed
    3. 可重复读 repeatable=read
    4. 串行化 serializable

  4. 数据库索引的作用

    1. 提高查询速度
    2. 确保数据的唯一性
    3. 可以加速表和表之间的连接，实现表和表之间的参照完整性
    4. 使用分组和排序子句进行数据检索时，可以减少分组和排序的时间
    5. 全文检索字段进行搜索优化。
    6. 分类：
      1. 主键索引PRIJMARY KEY：某一个属性组能唯一标识一条记录
      2. 唯一索引 UNIQUE：避免同一个表中某数据列中的值重复。
        1. 与主键索引的区别：
          1. 主键索引只能有一个
          2. 唯一索引可以有多个
      3. 常规索引 INDEX：快速定位特定数据
      4. 全文索引 FULLTEXT：快速定位特定数据
  5. 什么叫触发器。
    1. 是一种特殊的存储过程。
    2. 触发器主要是通过事件进行触发而被执行的，而存储过程可以通过存储过程名字而被直接调用。
    3. 触发器的主要作用就是器能够实现由主键和外键所不能保证的复杂的参照完整性和数据的一致性。

    1. 选取合适的字段属性，在可能的情况下，尽量将字段设置为not NULL，这样查询的时候就不用和NULL值进行比较。
    2. 使用连接JOIN来代替子查询
    3. 使用联合UNION来代替手动创建的临时表

    5. 锁定表
    6. 使用外键
    7. 使用索引
    8. 优化查询语句
  8. 数据库三大范式
    1. 第一范式：最基本的范式。如果数据库表中所有的字段都是不可分解的原子值，就说明该数据库表满足了第一范式
    2. 第二范式：在第一范式的基础上更进一层，第二范式需要确保数据库表中的每一列都和主键相关，而不能只与主键的某一部分相关。、
    3. 第三范式：需要确保数据表中的每一列数据都和主键直接相关，而不能间接相关。

JavaWeb

  1. 什么是MVC
    1. MVC全名是Model ，View，Controller是模型，视图，控制器的缩写。是一种软件设计典范，用一种业务逻辑，数据，界面显示分离的方法组织代码，将业务逻辑聚集到一个部件里面。
  2. JSP和servlet的异同点
    1. 相同点：JSP的本质是servlet，JSP是在servlet的基础上推出的，JSP具有Servlet的功能。
    2. 不同点：
      1. JSP编译后就成了Servlet（JSP的本质就是Servlet，JVM只能识别Java类，不能识别JSP的代码，Web容器将JSP编译成能够识别的java类）
      2. JSP更擅长表现与页面的显示，servlet更擅长于逻辑控制
      3. Servlet没有内置对象，JSP的内置对象都是必须通过HTTPServletRequest对象，HTTPServletResponse对象以及HTTPServlet对象获得
  3. JSP的九大内置对象
    1. request,response,pageContext,session,application,page,config,out,exception
  4. servlet的生命周期
    1. web容器加载servlet，生命周期开始。通过调用servlet的init（）方法进行servlet的初始化。通过调用service()方法实现，根据请求的不同调用不同的doGet/post方法。结束服务，web容器调用servlet的destroy方法。
  5. servlet是单例吗。线程安全吗
