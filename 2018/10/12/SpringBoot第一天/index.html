SpringBoot第一天

发表于 2018-10-12 | 分类于 框架

SpringBoot

SpringBoot是为了简化Spring应用的创建，运行，调试，部署等而出现的，使用它可以做到专注于Spring应用的开发，而无需过多关注XML的配置。

  1. 微服务：购物模块，秒杀的时候（流量很大），整个系统分割（功能模块分割），单独部署在一台服务器上，多服务构成一个系统，服务和服务之间可以数据共享，SpringBoot的底层是SpringMVC+Spring，上一层SpringCloud。
  2. SpringBoot默认使用Tomcat作为服务器【内置的】，，服务器netty【响应式的，非阻塞】
  3. 优势：简单、快速、方便。一般来说，搭建一个SpringWeb项目，步骤基本如下：
    1. 配置web.xml加载Spring和SpringMVC前端控制器
    2. 配置数据库连接、配置Spring事务
    3. 配置加载配置文件的读取，开启注解
    4. 配置日志文件log4j
    5. 配置之后部署Tomcat调试

SpringBoot2.0新特性

  1. 编程语言方面：仅支持jdk1.8，并且支持函数式编程语言，如kotlin
  2. 底层框架方面：Spring framework5.0大量使用了java8特性，并支持kotlin函数。其很多底层实现都是使用kotlin函数编写的。Lambda表达式
  3. 全新特性：webFlux替代SpringMVC，它是对SpringMVC做的一种补充，传统的SpringMVC是用过注解和标签来产生相应的请求映射以及响应。webFlux同样兼容这种方式，同时还提出了新型的声明式函数编程

Spring data jpa

  1. Spring data是Spring提出的一个持久层框架，也是目前的一种趋势。
  2. JPA：JAVA Persistence API定义了一系列对象持久化的标准，目前实现这类规范的产品有Hibernate，TopLink等
  3. Spring data jpa就是Spring对JPA的一种实现技术，是Spring和Hibernate的整合。Hibernate是一个完全的ORM模型，完全的面向对象的语言，可以不需要理解mysql的语法
